# Security Policy

## 安全特性

本项目实现了以下安全措施：

### 1. 配置安全

- **文件权限**: 配置文件 (`.cosrc`) 自动设置为 `600` 权限（仅用户可读写）
- **密钥脱敏**: `config show` 命令显示密钥为 `***`，不会泄露到控制台
- **路径防护**: 配置文件路径必须在当前目录或用户主目录下，防止路径遍历攻击

### 2. 输入验证

- **文件路径验证**: 上传前验证文件路径，确保：
  - 文件存在且为普通文件（非目录、非特殊文件）
  - 文件路径被解析为绝对路径，防止符号链接攻击
  - 文件大小不超过 5GB 上限

- **路径遍历防护**: `--config` 参数只允许指向当前工作目录或用户主目录下的文件

### 3. 依赖安全

```bash
# 检查依赖漏洞
npm audit

# 自动修复漏洞
npm audit fix
```

## 报告安全问题

如发现安全问题，请：

1. **不要** 在公开 Issue 中披露
2. 私下联系项目维护者
3. 提供详细的复现步骤

## 已知问题

### 依赖警告

`cos-nodejs-sdk-v5` 依赖的某些包存在已知漏洞（主要来自 `request` 等已弃用的依赖）。这些漏洞：
- 不影响核心上传功能
- 需要等待腾讯云 SDK 更新
- 可通过 `npm audit fix` 尝试修复

### 运行时警告

使用完整版 (`upload.js`) 时可能出现 `punycode` 弃用警告：
```
(node:xxx) [DEP0040] DeprecationWarning: The `punycode` module is deprecated
```

- **原因**: 依赖的 `tough-cookie` 使用了 Node.js 内置的已弃用模块
- **影响**: 仅警告，不影响功能
- **解决**: 使用 `node --no-deprecation scripts/upload.js` 运行

## 安全使用建议

1. **密钥管理**
   - 不要将密钥提交到版本控制
   - 使用环境变量或配置文件（已设置 `.gitignore` 忽略 `.cosrc`）
   - 定期轮换密钥

2. **文件上传**
   - 确保上传的文件不包含敏感信息
   - 大文件上传时注意网络环境安全

3. **权限设置**
   - 配置文件自动设置为 600 权限
   - 如需手动设置: `chmod 600 ~/.cosrc`
